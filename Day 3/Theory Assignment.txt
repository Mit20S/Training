1. Client Side Validation------
It is the user input validation that takes place on the client side.
It is important to ensure all required form controls are filled out.
Client-side validation is an initial check and an important feature of good user experience; by catching invalid data on the client-side, the user can fix it straight away. 
Client side validation is too easy to pass.


2. Advantages of Performing Validation ----
Increased Output
Optimised processes
Assured quality of product
No resources on the server side will be wasted.


3. QueryString ------
A query string is the portion of a URL where data is passed to a web application and/or back-end database.
The query string is whatever follows the question mark sign ("?") in URL

4. <input type="hidden">------
A hidden field lets web developers include data that cannot be seen or modified by users when a form is submitted.
Defines a field within a form that is not visible to the user.
To obtain information like IP address hidden tag can be used. Information that needs to be obtained but the user does not need to know. 
This may include security tokens, or the name of the relevant row in the database. The user does not need to see this data. 

5. Differnce between <submit /> and <button></button>------
Submit: The button submits the form data to the server. This is the default if the attribute is not specified, or if the attribute is dynamically changed to an empty or invalid value.
Button: The button submits the form data to the server. It is a simple button which is used to perform some operation by using javascript.
